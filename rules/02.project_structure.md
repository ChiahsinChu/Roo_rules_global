## 2. Directory Specifics

### `src/` (Source Code)

- **Purpose:** Contains all production-grade code.
- **Organization:**
- Group files by **feature/module**, not just file type.
- **Python:** Must contain `__init__.py` to define packages.
- **C++:** Headers (`.hpp`) and source (`.cpp`) should act as paired units.
- **Fortran:** Module files (`.f90`) reside here. Avoid standalone files unless necessary.

### `tests/` (Quality Assurance)

- **Purpose:** Automated tests to verify logic.
- **Rules:**
- Mirror the folder structure of `src/` (e.g., `src/calc/energy.py` â†’ `tests/calc/test_energy.py`).
- **Python:** Follow `pytest` conventions (prefix files and functions with `test_`).
- **C++:** Use GoogleTest or Catch2 conventions.
- **Fortran:** Use a standard testing framework (like `test-drive`) or custom driver programs.

### `examples/` (Usage Demos)

- **Purpose:** Runnable scripts for users to learn the software.
- **Structure:** **Every example must be contained in its own subdirectory.**
- **Naming:**
- Folders: `01_basic_usage/`, `02_advanced_simulation/`.
- Entry Point: `run.py`, `main.cpp`, or `driver.f90` inside the folder.
- **Requirements:**
- Each folder should be self-contained.
- Optional: Include a `README.md` inside the folder if the example requires specific inputs or data files.

### `docs/` (Knowledge Base)

- **Purpose:** Human-readable documentation.
- **`docs/agents/` (Vibe Coding):**
- **Purpose:** Specialized context for AI Agents to maintain project continuity ("Vibe Coding").
- **Required Files:**
- `AGENTS.md`: The "Brain" file containing project context, current focus, and memory.
- `todo.md`: A granular checklist of immediate tasks.

## 3. Root Level & Build Configuration

### Python Projects

- **Config:** `pyproject.toml`.
- **Dependencies:** `requirements.txt` or `environment.yml`.
- **Linting:** `.pylintrc` or `ruff.toml`.

### C++ / Fortran Projects

- **Build System:** `CMakeLists.txt` (Modern CMake 3.15+).
- **Build Output:** `build/` (Must be present in `.gitignore`).
- **External Libs:** Place manual dependencies (not managed by system package managers) in `external/` or `vendor/`.

## 4. General Rules

- **Output:** Generated artifacts (plots, logs, binaries) must go into a `_output/` or `build/` directory, which is ignored by git.
- **Case Sensitivity:** Respect the casing defined in [01.formatting-rules.md](./01.formatting-rules.md) for filenames.
